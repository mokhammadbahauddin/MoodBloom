# MoodBloom - Workspace Instructions

You are assisting with the development of "MoodBloom", a student wellness tracker app.

## Project Memory & Tracking
- **PRD**: See `./PRD.md` for the product requirements and current feature set. You must keep this file up to date when introducing new features.
- **Update Log**: See `./PROJECT_TRACKING.md`. You MUST append a new entry to this log whenever you complete a task, fix a bug, or make architectural changes. This acts as your memory of every update.

## Tech Stack Rules
- **UI**: React 19, Tailwind CSS 4, Motion, Lucide-React.
- **State**: Zustand with persist middleware.
- **Cloud**: Firebase (Auth, Firestore).
- **AI**: Node.js Express server acting as a proxy to Gemini API (`server.ts`).

When asked to update the project, refer to `PRD.md` and `PROJECT_TRACKING.md` to maintain context.
